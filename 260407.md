## Theorie
Man kann mit hilfe von Text-Editoren wie VSCode etc. Shell Scripts erstellen die man dann in der Linux shell ausführen kann dabei gibt es verschiedene Regel die man befolgen muss und verschiedene Anwendungen von SYNTAX damit es dann auch Funktioniert.

**TEST SHELL SKRIPT:**

```sh
#!/bin/env bash

# Hallo Welt

# Shell Script = Textdatei mit shell Befehlen
# Befehle ergeben das gleiche Ergebnis wie wenn in dernshell ausgeführt

echo $MYVAR Welt
echo Guten Morgen

FILE_NAME=test.txt
DIR_NAME=mydir
# mkdir $DIR_NAME
echo "Irgendein Text" > $DIR_NAME/$FILE_NAME

# double quotes
NAME="Denis"
TEXT="Hallo $NAME"
echo $TEXT

# single qoutes
TEXT2='Hallo $NAME'
echo $TEXT2

# curly braces {}

FILE=myname
echo "${FILE}_001.txt"
echo "${FILE}_002.txt"

```

Wie man sieht kann man verschiedene Variablen für Sachen anlegen und dann verwenden jedoch muss man auf die Quotes aufpassen.

---
### Variable anlegen
Man kann ganz einfach Variablen erstellen z.B.: `NAME=myname`

### Double Quotes
Verwendet man wenn ein Text ein `LEERZEICHEN` beinhaltet, man kann innerhalb der Quotes auch Variablen benutzen

### Single Quotes
Ähnlich wie die **Double Quotes** nur die Variablen werden nicht erkannt

### Curly Braces
Verwendet man damit man sagt dass z.B.: FILE ein Variable ist wenn man es für eine Datei verwendet wird wie z.B.: `.txt` etc.

## 2. Übung Begrüßung

**Script:**

```sh

#!/bin/bash

echo "Wie heißt du?"

read NAME

echo "Hallo $NAME"
echo "Schön, dass du da bist."
```

mit `read` kann man abfragen und dann eingeben, dann wird die Variable befüllt

## 3. Übung Zeilenumbruch

**Script:**

```sh
MSG=" \n Name: Denis \n Klasse: 3AHITS \n Raum: ..."

echo $MSG
```

**Ausgabe:**
```sh
┌──(kali㉿kali)-[~/260324]
└─$ ./shell.sh

 Name: Denis 
 Klasse: 3AHITS 
 Raum: ...
```

Wie beim programmieren kann man mit `\n` ein zeilenumbruch machen

## 4. Übung admin

```sh

USER=admin

echo ${USER}_backup
echo ${USER}_2025

```
man braucht die Curly Braces `{}` weil die Shell sonst vermutet das du eine Variable `USER_backup` ausgeben willst.

## 5. Übung log files

```sh
DIR="/var/log/"

echo "${DIR}nginx"
echo "${DIR}apache"
```
Eigentlich genau das selbe wie bei Übung 4 nur das mit die Double Quotes nutzen muss

## 6 Übung backup

```sh
BASE="backup"
DATE="2026-01-16"

echo "${BASE}_${DATE}.tar"
echo "${BASE}_${DATE}.tar.gz"
```

Genau wie bei Übung 5 muss man darauf achten das man die Double Quotes und Curly Braces benutz
